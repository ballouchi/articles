Troubleshoot Problems with Audio Using PulseAudio Volume Control

If you’re experiencing problems with audio in Linux, chances are you can solve them using the PulseAudio Volume Control application. This is a simple GUI-based audio control program that allows you to control the settings for sound input, output, recording, and more. PulseAudio Volume Control comes pre-installed with certain Linux distros, and is also available as a Flatpak app.

Installation of PulseAudio Volume Control

If PulseAudio Volume Control did not come pre-installed with your Linux distribution, you can install it manually with the following commands.

Ubuntu/Debian

sudo apt-get install pavucontrol

Fedora

sudo dnf install pavucontrol

openSUSE

sudo zypper install pavucontrol

Or, after doing the preliminary installation and setup for Flatpak, you can install the Flatpak for PulseAudio Volume Control with this command:

sudo flatpak install flathub org.pulseaudio.pavucontrol

Listing Audio Devices

Before you get started, open a terminal window and type the command:

aplay --list-devices

This will give you a list of the hardware audio devices that are currently connected to your computer. If you connect new devices, repeat the command to get a new list. PulseAudio Volume Control uses different device names than the one displayed by this command, but the list can still help you get an idea of which devices your system is currently using.

Starting PulseAudio Volume Control

In most Linux distros, you can type ALT+F2 to open a command box. Then simply type pavucontrol in the box and hit enter. You can also run pavucontrol from the terminal emulator. Many distros also embed PulseAudio Volume Control in the menus, usually under the “Sound” category. Some desktop environments, like Unity, or XFCE with the Whisker Menu, allow you to search for the program you want to run. Check the documentation for your Linux distro or desktop environment to find the correct method to start programs from the program menus.

Troubleshooting Problems with Playback

You can troubleshoot problems with playback using PulseAudio Volume Control in several ways. The first is to click the Playback tab, and adjust the settings there.

Underneath each category, there is a slider which you can use to adjust the volume from silence to above 100%. 100% is typically regarded as the ideal adjustment for any audio output, but it may be too soft or too loud for your particular audio output. If there is noise or buzzing, try adjusting the slider to the left. If the volume is too low even when all audio levels are at maximum, try adjusting the slider to the right.  If there is no sound, uncheck the mute symbol, which appears as a volume icon with a red X. When the mute option is active, this symbol appears surrounded with a grey square. When the mute option is not active, the symbol appears surrounded by white.

The bottom bar below the word “Silence” will fill with color if your audio device is working. If it is not working, it will remain greyed out. While the playback is running, watch this bar to see the levels for your playback device.

You can also choose to change the audio stream that PulseAudio Volume Control uses. For example, on my system, the above Playback tab offers a choice between “Built-in Audio Analog Stereo” and “FFT Based Equalizer on Built-in Audio Analog Stereo”.

If necessary, beside “Show”, you can change the drop-down to All Streams or Virtual Streams to see more options.

Troubleshooting Problems with Recording

If you’re having issues with your microphone or recording software, click the Recording tab. Here, any applications that are accessing your microphone will appear. One of the problems that can cause Linux to have difficulty capturing a recording is that two or more applications are vying for the microphone. If this is happening, you may see two or more applications in the Recording tab. If this is the case, try closing the applications you are not currently using (you can use the kill or killall commands, for example) before using PulseAudio Volume Control to troubleshoot your issue.

In the above figure, two different applications are trying to access the microphone. The top application is Firefox and the bottom one is Audacity. This sort of conflict can cause a microphone to work erratically, or not at all.

Another problem with recording is input. You must ensure that PulseAudio is set up to use the microphone you want before recording. For example, if you have an external microphone plugged in by USB, and also an internal microphone (such as on a laptop), be sure to select the external USB microphone as your input device, or the computer may revert by default to using your internal microphone. However, if you do want to use the internal microphone, you can use the drop-down to select it. On my laptop, for example, it appears as “Monitor of Built-in Analog Stereo”.

Also, Linux has poor compatibility with many microphones. If you continue to have a problem with recording, you can try swapping out microphones and see if one works better.

Troubleshooting Speakers and Headphones

Click the Output tab.  The output devices that are active on your computer will appear here. As in the Playback tab, moving the slider to the right increases the volume, and moving it to the left decreases the volume. The volume icon with the red X mutes the application, the lock icon locks channels together, and the green icon sets the output device as the default device. Deactivating the lock channels option allows you to control the volume of each speaker (or each side of your headphones) separately. This is especially helpful if one channel is too loud or too soft.

The bottom bar, underneath the word “Silence” will fill with color if your audio output device is working.  The “Advanced” field allows you to change the latency offset. This setting is often useful for audio that is lagging, or video that is out of sync.

In the above figure, “Built-in Audio Analog Stereo” is a hardware device, and “FFT based equalizer on Built-in Audio Analog Stereo” is a virtual device. The volume levels are low because the overall volume of the Linux system is turned down. On my system, turning up the volume levels in PulseAudio Volume Control turned up the overall volume levels for the computer.

If you see activity in the bottom bar within these fields but you don’t hear any sound, there may be a problem with your speakers or headphones. You can try swapping out your speakers or headphones to see if one set works better. (Also, it may be helpful to plug your speakers or headphones into a different computer or different OS to make sure they’re working properly.) You can also try swapping out audio cards as an absolute last resort, but computer circuits are extremely sensitive to static electricity, so handle the cards carefully, set them on an anti-static surface, and always hold them by the edges.

Troubleshooting Input Devices

If you’re having a problem with your microphone, you’ll need to manage your input devices in PulseAudio Volume Control. Click the Input Devices tab. Here you will see a list of your active input devices, typically a microphone, internal or external, or a device that’s plugged in to a line-in port. Moving the volume sliders to the right will increase the gain on your microphone, and moving them to the left alternately decreases the gain. It’s best to listen to playback of your recording before trying to adjust these sliders. If you feel that the volume of your recording is too low, move the sliders to the right. But if you find a lot of noise such as distortion, cracking, or popping on your recording, move them to the left.  The “Base” level is a pretty good level for most microphones, so always move your sliders there first before making these adjustments.

Deactivating the lock channels option may be useful if you have a stereo microphone. The mute button is especially helpful if you have multiple microphones connected to your computer, but want to manually control which ones are recording at any given time.

The Advanced option, again, is for setting latency offset.

In the Show field, you can choose to display All Input Devices, All Except Monitors, Hardware Input Devices, Virtual Input Devices and Monitors.

Troubleshooting Audio Playback Devices

Click the Configuration tab. Here is where you can select the audio device you want to use for playback. When you select your device here, it will show up in the other tabs.

You may need to select your audio device here if you are using external speakers, or another external sound device. For example, if you connect your computer to your HDTV by HDMI, you will have to change the profile to one of the HDMI options to hear the audio through the speakers on the HDTV. If you don’t, the video may appear on the screen, but the audio will continue to flow through your default device (most likely the sound card and speakers on your PC).

In most cases, the best way to troubleshoot this problem in the Configuration tab is simple trial and error. Just keep selecting different profiles in the drop-down menu until you find one that works.

If troubleshooting with PulseAudio Volume Control fails to solve your audio problem, try booting Linux in a live session on CD, DVD, or USB, and see if the problem recurs. If it does recur, you may have a hardware problem, whereas if it doesn’t, the problem is likely software-related.
