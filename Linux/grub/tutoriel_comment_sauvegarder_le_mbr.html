Comment sauvegarder son MBR ?

Ce tutoriel vous permettra de sauvegarder le MBR de votre PC (sur cl USB, sur email, ou autre), pour pouvoir le restaurer si besoin est.

C'est la premire chose faire avant d'installer Ubuntu si vous suspectez votre PC d'tre tatou, ou si vous souhaitez (r)installer Windows.

Qu'est-ce que le MBR ?

Pour les PCs partitionns en Ms-Dos seulement.

Le MBR est le premier secteur du disque. On trouve aussi au dbut de chaque partition tendue un EBR qui est son quivalent pour dcrire les partitions logiques embotes.

Le Master Boot Record, zone d'amorage principale, est une zone de 512 octets dcoupe ainsi :

  • les 446 premiers octets constituent le bootloader, code excutable qui permet l'accs initial aux informations du disque ainsi que les messages d'erreur.
  • les 64 suivant contiennent l'arrangement de votre disque dur : la table des partitions primaires (64 octets, soit 16 octets pour chacune des partitions primaires possibles)
  • et enfin deux octets "magiques" 55 AA indiquant que le secteur est amorable.

Lors du dmarrage de l'ordinateur, le BIOS scrute successivement les diffrents priphriques la recherche de cette signature pour charger le secteur en mmoire.

Dans certains cas (manipulation hasardeuse des partitions, extinction brutale lors d'un redimensionnement, virus de MBR attrap avec un autre OS) il arrive de perdre ces informations. Dans le meilleur des cas, une ou plusieurs partitions sont inaccessibles. Dans le pire des cas, le disque apparat comme "non format". Il `suffit` alors d'crire nouveau ce MBR en lieu et place de celui qui est corrompu. Encore faut-il en avoir fait une copie avant. Nous vous proposons donc ici de sauvegarder le MBR au complet, et les commandes de restauration permettant de ne restaurer que le bootloader, que la table de partitions, ou tout le MBR. Il est vident qu'un fichier de sauvegarde de ce type ne doit pas tre conserv sur le disque qu'il faudrait "rparer", mais sur un autre support de sauvegarde (CD, cl USB, etc).

Procdure de sauvegarde du MBR et de la table de partition complte

Les 2 mthodes ci-dessous peuvent tre ralises sans installer Ubuntu, aussi bien depuis un live-CD qu'une session Ubuntu installe.

Graphiquement

  1. il suffit de lancer le logiciel Boot-Repair, puis de cliquer sur "Options avances", puis sur le bouton "Sauvegarder les tables de partitions".
  2. puis enregistrez le fichier ZIP sur cl USB ou envoyez-le-vous par courriel. Ce fichier ZIP contient les sauvegardes de MBR (fichiers .img) et de tables de partitions (fichiers .dmp) de tous les disques de votre ordinateur.

En ligne de commande

  1. pour sauvegarder le MBR : saisissez dans un terminal la commande suivante qui va crer un fichier nomm mbr.img dans votre Dossier Personnel1): 
    sudo dd if=/dev/sda of=~/mbr.img bs=512 count=1
  2. pour sauvegarder la table des partitions complte (avec les partitions logiques) :
    sudo sfdisk -d /dev/sda > table_partitions_sda_$(date +%Y%m%d).dmp
  3. puis enregistrez ces 2 fichiers sur cl USB ou envoyez-les-vous par courriel.

Remarques:

  • Si vous souhaitez sauver le MBR d'un autre disque que sda, n'oubliez pas d'adapter la commande.
  • Si votre ordinateur possde plusieurs disques, rptez la procdure pour chaque disque.

Procdure de restauration du MBR

Cette procdure restaure le MBR sans restaurer la table de partition.

  1. Bootez votre ordinateur sur un Live CD Ubuntu2)
  2. placez votre sauvegarde dans votre dossier personnel, renommez-la en mbr.img , puis saisissez dans un terminal:
sudo dd if=~/mbr.img of=/dev/sda bs=446 count=1

Procdure de restauration de la table de partitions

Attention, restaurer votre table de partition peut vous empcher d'accder vos partitions si elles ont t modifies aprs avoir effectu la sauvegarde.

Si vous disposez d'une sauvegarde de table de partitions complte (.dmp)

Pour restaurer la table des partitions complte ( partir d'un fichier .dmp cr via Boot-Repair ou sfdisk, cf procdures ci-dessus) : bootez votre ordinateur sur un Live CD Ubuntu3), placez votre sauvegarde dans votre dossier personnel, renommez-la en table_partitions.dmp , puis saisissez dans un terminal: 

sudo sfdisk /dev/sda < table_partitions.dmp

Si vous disposez d'une sauvegarde de MBR (.img)

  • Pour ne restaurer que la table de partition primaire ( partir d'un fichier .img cr via Boot-Repair ou dd, cf procdures ci-dessus) : bootez votre ordinateur sur un Live CD Ubuntu4), placez votre sauvegarde un endroit accessible comme par exemple votre dossier personnel5), renommez-la en mbr.img , puis saisissez dans un terminal:
sudo dd if=~/mbr.img of=/dev/sda bs=1 skip=446 seek=446 count=66
  • Pour restaurer le MBR et la table de partition primaire: bootez votre ordinateur sur un Live CD Ubuntu6), placez votre sauvegarde dans votre dossier personnel, renommez-la en mbr.img , puis saisissez dans un terminal:
sudo dd if=~/mbr.img of=/dev/sda bs=512 count=1
