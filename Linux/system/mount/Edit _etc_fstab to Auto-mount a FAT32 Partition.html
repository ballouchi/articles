<SOLVED> Edit /etc/fstab to Auto-mount a FAT32 Partition?

I don't have a FAT32 partition myself, but am trying to instruct someone how to edit /etc/fstab to auto-mount one. I'm confused by the instructions I'm seeing on various sites. Here are two examples:

1. From Mounting Windows Partitions -- Community Help Wiki, it gives the following example.
UUID=<UUID> /media/<mountpoint> vfat defaults,user,exec,uid=1000,gid=100,umask=000 0 0

2. From Fstab -- Community Help Wiki, it gives this example.
/dev/hda2 /media/data1 vfat defaults,user,exec,uid=1000,gid=100,umask=000 0 0

That doesn't make sense to me. Why are they using "gid=100" instead of what I think it should be -- gid=1000?

Re: Edit /etc/fstab to Auto-mount a FAT32 Partition?

Post by altair4 Thu May 08, 2014 7:09 am

Actually, with a umask of 000 both uid and gid are unnecessary since that umask gives everyone read / write access anyway.

Anyway, once upon a time all users were added to the "users" group ( gid=100 ) out of habit. NTFS and FAT32 partitions in order to be accessed by all local users included that gid and the line in fstab was usually written this way:
UUID=<UUID> /media/<mountpoint> vfat defaults,user,exec,uid=1000,gid=100,umask=007 0 0
The resulting mounted partition have permissions of 770 so user 1000 and all local users ( everyone who was a member of the "users" group ) would have access but no one else.

Side note: You really don't need the "user,exec" options since in this context they won't do anything.

Re: Edit /etc/fstab to Auto-mount a FAT32 Partition?

Post by altair4 Thu May 08, 2014 3:12 pm

Yes, As long as you remember to add whoever you want to allow access is a member of the "users" group since they are not that by default.
EDIT: Sorry posted past each other.

The umask=007 vs umask=000 is up to you. One restricts it to members of a specific group and the other makes it available for everyone. If there's only one user it makes no difference.

Actually if there's only one user you can just take possession of the mounted partition and remove any reference to gid:

Code: Select all

UUID=<UUID> /media/<mountpoint> vfat defaults,uid=1000,umask=007 0 0
